# Client Architecture

## System Design
The Client is designed as a **Database Browser**. Unlike traditional Single Page Apps (SPAs) that fetch data per-page from an API, this application downloads the *entire* dataset upfront (or typically via `timeline.json`) and loads it into **IndexedDB**.

### Why Local-First?
1.  **Performance**: Filtering/Sorting 5,000+ events happens instantly in-memory/IDB without network roundtrips.
2.  **Graph Analysis**: Physics simulations (D3 Force) require access to all nodes/links simultaneously.
3.  **Resilience**: The app works offline once loaded.

## Core Components

### 1. The Data Layer (`src/db`)
-   **Dexie.js**: We use Dexie to wrap IndexedDB.
-   **Schema**:
    -   `events`: The core timeline events (indexed by date, type, etc.).
    -   `validations`: Validation records synced from backend.
-   **Loader Pattern**: On startup, `loader.ts` checks if the remote `timeline.json` is newer than the local DB. If so, it nukes and rebuilds the local DB.

### 2. The Visualization Layer (`src/components/visualization`)
We use a **React-wrapped D3** pattern.
-   **React** handles the Container DOM, Resize Observation, and Props (Data/Settings).
-   **D3** handles the internal SVG manipulation via `useEffect`.
-   **Data Processing**: `useGraphData.ts` hook runs expensive graph transformations (Link generation, Metric calculation) inside `useMemo` to prevent D3 re-renders on simple UI interactions.

### 3. The Validation System
-   **Source of Truth**: `validations.json` (generated by backend build).
-   **Frontend Integration**:
    -   `syncValidations()` function loads this into `db.validations`.
    -   `ValidationBadge` component queries this table to show Verified/Rejected status.
    -   Network nodes are colored based on this status integration.

## Directory Map
```
src/
  ├── components/       # UI Components
  │   ├── common/       # atomic UI (Badges, Buttons)
  │   ├── editor/       # Event Editing Forms
  │   └── visualization/# D3 Views (Graph, Matrix, etc.)
  ├── db/              # Database Definition & Sync Logic
  ├── hooks/           # Shared React Logic
  ├── schemas/         # Zod Definitions (Shared with Backend conceptually)
  ├── types/           # TypeScript Interfaces
  └── utils/           # Pure Logic (Math, Graph Algos)
```
